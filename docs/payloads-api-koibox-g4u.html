GET /api/clientes/?movil={telefono}

1. Buscar cliente existente

Buscamos primero por telefono, si no hay resultado buscamos por email. Esto evita crear duplicados.

Request
// Busqueda por telefono (prioridad)
GET /api/clientes/?movil=%2B34612345678

// Si no hay resultado, busqueda por email
GET /api/clientes/?email=maria%40ejemplo.com

Headers: {
  "X-Koibox-Key": "{API_KEY}"
}
Response (si existe)
{
  "count": 1,
  "results": [{
    "id": 12345,         // ← Este es el ID interno que usamos
    "nombre": "Maria",
    "apellido1": "Garcia Lopez",
    "email": "maria@ejemplo.com",
    "movil": "+34612345678"
  }]
}
POST /api/clientes/

2. Crear cliente nuevo

Si la busqueda no devuelve resultados, creamos el paciente. Separamos nombre y apellido automaticamente.

Request Body
{
  "nombre": "Maria",
  "apellido1": "Garcia Lopez",
  "email": "maria@ejemplo.com",
  "movil": "+34612345678",
  "sexo": "M",              // H = hombre, M = mujer
  "notas": "Quiz resultado: alopecia androgenetica...",
  "origen": "w",             // w = web
  "provincia": 680           // 680=Madrid, 697=Murcia, 718=Pontevedra
}
Response
{
  "id": 12345,              // ← ID interno para operaciones posteriores
  "nombre": "Maria",
  "apellido1": "Garcia Lopez",
  ...
}
PATCH /api/clientes/{koibox_client_id}/

3. Actualizar cliente existente

Si el paciente ya existe, solo actualizamos las notas, anadiendo la informacion del quiz sin sobreescribir.

Request Body
PATCH /api/clientes/12345/

{
  "notas": "{notas_existentes}\n---\n[G4U Quiz] {nuevas_notas}"
}
ⓘ Concatenamos las notas nuevas con las existentes usando un separador ---, nunca sobreescribimos.
POST /api/agenda/

4. Crear cita

Creamos la cita vinculada al paciente usando su ID interno de Koibox. El titulo incluye el nombre del paciente.

Request Body
{
  "titulo": "Diagnostico Capilar - Maria Garcia",
  "fecha": "2026-03-28",
  "hora_inicio": "10:00",
  "hora_fin": "10:30",
  "user": 30257,                    // ID empleado/hueco de agenda
  "servicios": [103385],            // Primera Consulta Diagnostico (0 EUR)
  "cliente": 12345,                 // ← ID interno Koibox del paciente
  "notas": "Reserva desde quiz online — BONO DIAGNOSTICO PAGADO"
}
Response
{
  "id": 67890,              // ← ID de la cita (para cancelar/reagendar)
  "titulo": "Diagnostico Capilar - Maria Garcia",
  "fecha": "2026-03-28",
  "hora_inicio": "10:00",
  "hora_fin": "10:30",
  ...
}
⚠ Las notas varian segun si el paciente ha pagado el bono diagnostico o no. Se verifica contra GHL antes de crear la cita.
PATCH /api/agenda/{koibox_appointment_id}/

5. Cancelar cita

Se cancela cambiando el estado a 5. Se usa cuando el paciente no confirma en 48h o cancela manualmente.

Request Body
PATCH /api/agenda/67890/

{
  "estado": 5                       // 5 = cancelada
}
GET /api/agenda/?fecha={fecha}&user={employee_id}

6. Consultar disponibilidad

Consultamos las citas existentes para una fecha y calculamos los huecos libres del lado del cliente.

Request
GET /api/agenda/?fecha=2026-03-28&user=30257
Logica de disponibilidad
// Calculamos huecos libres client-side:
// - Horario: 09:00 - 20:00
// - Slots de 30 minutos
// - Excluimos slots que coinciden con citas existentes
// - Excluimos slots en el pasado (si fecha = hoy)
